Introduction

House prices can often be completely unpredictable and can be quite crazy so we aim to predict house prices using the Boston Housing dataset. The dataset contains information on various factors affecting housing prices in Boston, such as crime rate, number of rooms per dwelling, and property tax rates. By understanding the relationships between these factors we can determine exactly how much effect each one has on house prices so we can have a better understanding of what goes into determining the price of a house. It might even let us get a house that is undervalued by the market.

What is regression?

Regression is a statistical technique used to model relationships between a dependent variable and some independent variables. Linear regression assumes a linear relationship between the dependent variable and the independent variables. It applies a line of best fit that tries to predict the data based off of certain weightings

Data Understanding

First, we should examine the exact features of the dataset that we are working with. Some of the features we have are CRIM (crime rate per capita), RM (average number of rooms per dwelling), LSTAT (percentage of lower status population), TAX (property tax rate per $10,000), and PTRATIO (pupil teacher ratio by town). We should make a heatmap so we can figure out exactly which features are correlated with each other and use them to draw understandings about house prices.

Pre-Processing

We will also have to pre process the data to ensure that it is clean and workable. This will ensure that are datas results are not obscured by random noise or other features in the data that we do not want to. We will replace the missing values with the mean values. We will also drop any duplicates to ensure no duplicates are present. We will also get rid of infinite values and strip the names so that we are not thrown off by anything

Experiment 1

In the first experiment, we use Linear Regression, which is one of the simplest and most commonly used algorithms for regression tasks. Linear Regression tries to find the best fitting linear relationship between the input features and the target variable. The model assumes that the relationship between features and the target is linear. We will use all of the features to predict our home value.

In[297]:
X = data.drop(columns=["medv"])  
y = data["medv"]

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, random_state=47)

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)
In[299]:
lr = LinearRegression()
lr.fit(X_train_scaled, y_train)
y_pred_lr = lr.predict(X_test_scaled)
rmse_lr = np.sqrt(mean_squared_error(y_test, y_pred_lr))
print(f"Linear Regression RMSE: {rmse_lr}")

plt.figure(figsize=(12, 6))
plt.scatter(y_test, y_pred_lr, alpha=0.5)
plt.plot([min(y_test), max(y_test)], [min(y_test), max(y_test)], '--k')
plt.xlabel("Real Prices")
plt.ylabel("Predicted Prices")
plt.title("Lin Regression Predictions vs Real Prices")
plt.show()

Linear Regression RMSE: 4.933914182912892

We have got a RMSE of 4.93 which means on average we are 4.93 off of the actual value with our prediction so in the context of house prices it is not a bad amount to be off by

Experiment 2

In this one, we improve the model by removing features that are too similar to each other. When two features are highly linked, it can confuse the model and make predictions less reliable. For example, if "crime rate" and population density are closely linke we might remove one to avoid redundancy.

We also adjust certain features to improve accuracy. Since "crime rate" has a skewed distribution, we apply a log to make it more balanced. Additionally, we introduce polynomial features to help the model capture patterns that arent purely linear.

In[300]:
poly = PolynomialFeatures(degree=2, include_bias=False)
X_train_poly = poly.fit_transform(X_train_scaled)

Experiment 3

In this experiment, we use Ridge Regression, which is similar to Linear Regression but includes regularization. Regularization prevents the model from relying too much on certain features by keeping the coefficients smaller which ends ups reducing the risk of overfitting.
